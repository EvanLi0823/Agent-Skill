## 课程结构

教程包含 **9 个章节和练习题**，分为三个难度级别：

### 🌱 初级部分

#### 第 1 章：基本提示词结构
- 提示词的组成部分
- 角色定义（system、user、assistant）
- 基本格式规范

**关键要点**：
```
System: 你是一个有帮助的助手。
User: 请帮我...
Assistant: 好的，我来帮你...
```

---

#### 第 2 章：清晰和直接
- 避免模糊表达
- 使用明确的指令
- 指定输出格式

**示例对比**：

❌ **不好的提示词**：
```
告诉我一些关于AI的事情
```

✅ **好的提示词**：
```
请用3个要点总结AI Agent的核心概念。
每个要点不超过50字。使用项目符号格式。
```

---

#### 第 3 章：分配角色（Role Prompting）
- 为 Claude 指定专业角色
- 影响回答风格和深度
- 设定专业背景

**示例**：
```
你是一位拥有20年经验的Python专家。
请用初学者能理解的方式解释装饰器。
```

---

### 🌿 中级部分

#### 第 4 章：将数据与指令分离
- 使用 XML 标签区分内容
- 避免提示词注入
- 清晰的结构化

**示例**：
```xml
<instructions>
总结以下文章的主要观点
</instructions>

<article>
{文章内容}
</article>
```

---

#### 第 5 章：格式化输出 & 为 Claude 代言
- 指定输出格式（JSON、表格、列表）
- 使用 prefilling 技术
- 控制响应风格

**Prefilling 技巧**：
```
User: 用JSON格式列出三种水果
Assistant: {
```

Claude 会继续完成 JSON 结构！

---

#### 第 6 章：预知（Thinking Step by Step）
- 让 Claude 展示思考过程
- 使用 `<thinking>` 标签
- 提高复杂推理准确性

**示例**：
```
在回答前，先在<thinking>标签内分析问题。
然后在<answer>标签内提供最终答案。
```

---

#### 第 7 章：使用示例（Few-Shot Learning）
- 提供 2-5 个高质量示例
- 示例应多样化
- 展示期望的行为模式

**示例**：
```
示例1：
输入：今天天气真好
输出：积极

示例2：
输入：我很失望
输出：消极

现在请分类：我对结果很满意
```

---

### 🌳 高级部分

#### 第 8 章：避免幻觉
- 要求引用来源
- 使用"我不知道"选项
- 验证事实性声明

**技巧**：
```
如果你不确定答案，请明确说"我不确定"
而不是猜测。如果可能，请引用具体来源。
```

---

#### 第 9 章：构建复杂提示词（行业用例）
- 多步骤任务分解
- 条件逻辑
- 错误处理
- 实际业务场景

**复杂提示词模板**：
```xml
<role>
你是一个客户服务AI助手
</role>

<task>
1. 分析客户问题
2. 确定问题类别
3. 提供解决方案
4. 如果无法解决，升级给人工
</task>

<guidelines>
- 始终保持礼貌
- 使用简单语言
- 提供具体步骤
</guidelines>

<customer_message>
{消息内容}
</customer_message>
```

## 核心原则总结

### 1. 清晰性原则
```
明确 > 模糊
具体 > 抽象
简洁 > 冗长
```

### 2. 结构化原则
```
使用标签组织内容
分离指令和数据
明确输出格式
```

### 3. 上下文原则
```
提供充分背景
包含相关示例
设定合适角色
```

### 4. 迭代原则
```
测试提示词
分析结果
不断优化
记录最佳实践
```

## 常见失败模式与解决方案

### 失败模式 1：输出不一致
**问题**：每次运行结果不同
**解决**：
- 提供明确的输出格式
- 使用示例约束行为
- 降低 temperature 参数

---

### 失败模式 2：遗漏关键信息
**问题**：Claude 忽略了重要细节
**解决**：
- 使用 XML 标签强调重点
- 明确列出所有要求
- 要求确认理解

---

### 失败模式 3：幻觉（虚构信息）
**问题**：Claude 编造事实
**解决**：
- 要求引用来源
- 允许"不知道"答案
- 提供事实性检查指令

---

### 失败模式 4：指令被忽略
**问题**：Claude 没有遵循所有指令
**解决**：
- 使用编号列表
- 分步骤明确说明
- 要求逐项确认

## 实用技巧集锦

### 技巧 1：使用 XML 标签
```xml
<instructions>
你的指令
</instructions>

<context>
背景信息
</context>

<examples>
示例
</examples>
```

**优势**：
- 结构清晰
- 避免混淆
- 易于调试

---

### 技巧 2：Prefilling（预填充）
```
User: 用JSON格式列出信息
Assistant: {
  "name":
```

**优势**：
- 强制特定格式
- 减少格式错误
- 提高一致性

---

### 技巧 3：Chain of Thought（思维链）
```
在给出答案前，请：
1. 分析问题
2. 列出相关信息
3. 逐步推理
4. 得出结论
```

**优势**：
- 提高准确性
- 可追踪推理
- 减少错误

---

### 技巧 4：Few-Shot Examples
```
这里是一些例子：

例1：
输入：{示例输入1}
输出：{期望输出1}

例2：
输入：{示例输入2}
输出：{期望输出2}

现在处理：{实际输入}
```

**优势**：
- 明确期望
- 展示格式
- 提高准确性
